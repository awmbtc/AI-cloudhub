# 对象存储厂商清单与适配顺序

> 调研约 2026-07。免费额度以各厂商控制台为准，会变动。  
> 产品目标：**Drive Map = 把对象存储定义成盘符（G: / /workspace）**。  
> **本仓库 = AI-cloudhub（Go 自研 Workspace OS），不是魔改 MinIO。**  
> 下表均为 **用户可接入的后端**；架构定稿见 [ARCHITECTURE.md](./ARCHITECTURE.md)。

---

## 1. 国际向：流行且有免费档 / 适合挂盘

| 厂商 | 产品 | 免费额度（约） | 出口流量 | S3 兼容 | 备注 |
|------|------|----------------|----------|---------|------|
| **Cloudflare** | **R2** | **10GB 永久**；写约 100 万次/月、读约 1000 万次/月免费 | **$0 全免** | ✅ | **首选**；免出口最适合 Agent 读写 |
| **Backblaze** | **B2** | 常见约 **10GB** | 有免费额度；经 CF 等联盟可免 | ✅ | 存储单价低，备份/盘符常用 |
| **Oracle** | **OCI Object Storage** | Always Free 对象类合计约 **20GB** | 免费档含较大出站 | ✅（兼容层） | 永久免费，注册门槛高 |
| **Amazon** | **S3** | 新客约 **5GB / 12 个月** | 贵 | ✅ 原生 | 用通用 S3 适配即可 |
| **Wasabi** | Wasabi | 基本无 free tier | 规则特殊 | ✅ | 付费低价，非免费优先 |
| **Supabase** | Storage | 约 **1GB** | 有限 | 偏自有 API | 小文件，后做 |
| **自建** | **MinIO** | = 用户自己的磁盘 | 用户带宽 | ✅ | 轻量自建 = 真免费容量 |
| 其他 | DigitalOcean Spaces 等 | 通常无永久免费 | 按量 | ✅ | 通用 S3 模板可顶 |

**海外优先级直觉：R2 > B2 > OCI > 通用 S3；自建 MinIO 始终 P0。**

## 2. 国内向：主流 + 免费 / 活动

| 厂商 | 产品 | 免费 / 羊毛（约） | S3 兼容 | 备注 |
|------|------|-------------------|---------|------|
| **七牛** | **Kodo** | 实名约 **每月 10GB 存储 + 10GB CDN 回源 + 读写请求额度**；上传流量常免 | 部分/需确认 | 国内少见的**持续月免** |
| **腾讯云** | **COS** | 新客常见 **50GB 标准 × 约 6 个月**；轻量有时赠对象额度 | ✅ 可选 | 国内主流 |
| **阿里云** | **OSS** | 新客常见 **40GB × 约 6 个月**；资源包活动多 | ✅ | 国内主流 |
| **又拍云** | 云存储 | 联盟约 **10GB + 15GB CDN**（需申请） | 自有 API | 审核制，后做 |
| **腾讯轻量等** | 附赠对象 | 部分套餐约 **50GB** 对象额度 | 视产品 | 可归入 COS/通用 |
| **华为 / 京东 / 青云等** | OBS 等 | 偶发新客包 | 多可 S3 | 通用 S3 先顶 |

**国内：七牛（月免）+ 阿里 OSS / 腾讯 COS（活动 + 正式用户）。**

## 3. 适配开发顺序（按此开搞）

### 批次 A — 当前进行中（P0）

| 顺序 | ID | 厂商 | 说明 |
|------|-----|------|------|
| **A1** | `s3` | **通用 S3** | 自定义 endpoint / region / path-style，通吃 AWS 与未知 S3 兼容站 |
| **A2** | `r2` | **Cloudflare R2** | 模板化 endpoint + account_id |
| **A3** | `minio` | **MinIO** | path-style 默认；用户自建 |
| **A4** | 挂载交付 | rclone 配置 | 映射成 `G:` 或 `/workspace` |

### 批次 B — 已完成

| 顺序 | ID | 厂商 |
|------|-----|------|
| B1 | `b2` | Backblaze B2 |
| B2 | `oss` | 阿里云 OSS（S3 兼容） |
| B3 | `cos` | 腾讯云 COS（S3 兼容） |

### 批次 C — 已完成（C1/C2）

| 顺序 | ID | 厂商 |
|------|-----|------|
| C1 | `qiniu` | 七牛 Kodo（S3 兼容） |
| C2 | `oracle` | Oracle OCI（S3 兼容层） |
| C3 | `upyun` / `supabase` / `wasabi` … | 按需 |

### 实现策略

```text
一批代码：S3 Compatible Driver
  → s3 / r2 / minio / b2 / oss / cos / qiniu / oracle（配置模板不同）

二批：厂商原生 API（若 S3 兼容不稳）
  → upyun / supabase
```
